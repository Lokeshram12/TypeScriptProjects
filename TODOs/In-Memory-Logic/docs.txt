## **3️⃣ Running TS code in Node (ESM)**

**Key concepts learned:**

* Node executes **JavaScript**, so TypeScript needs to be **compiled/transpiled** first.
* You used **ES6 modules** (`import`/`export`) instead of CommonJS (`require`/`module.exports`).
* `tsconfig.json` controls compilation, module system, and target JavaScript version.
* `package.json` with `"type": "module"` tells Node to treat `.js` files as ESM.

**Why:**

* ESM allows modern import/export syntax, which is **more standard**.
* `tsconfig.json` gives you control over how TS is converted to JS.

## **5️⃣ Dealing with ESM + ts-jest Issues**

**Problems you encountered & fixes:**

1. `Cannot use import statement outside a module` → Fixed by:

   * Setting `"type": "module"` in `package.json`
   * Using `ts-jest/presets/default-esm`
   * Adding `extensionsToTreatAsEsm: ['.ts']`

2. Importing `.ts` files → Fixed by enabling:

   * `"allowImportingTsExtensions": true` in `tsconfig.json`

3. Coverage only showing tested files → Fixed by:

   * Adding `collectCoverageFrom: ["src/**/*.ts"]` in Jest config

**Why:**

* Node and Jest need **matching module resolution and file extension rules**.
* Coverage requires Jest to know **all files in your project**, not just the ones touched by tests.
